# Structured Logging & Observability — 0.0.14

## P33-1: Structured Logging

The `tracing` crate is already a workspace dependency. All cloud-server and sync code uses `tracing::info!`/`warn!`/`error!` macros.

### Configuration

To enable JSON-structured logging, use `tracing-subscriber` with the `json` feature:

```rust
use tracing_subscriber::fmt::format::JsonFields;
use tracing_subscriber::EnvFilter;

tracing_subscriber::fmt()
    .with_env_filter(EnvFilter::from_default_env())
    .json()
    .with_file(true)
    .with_line_number(true)
    .with_target(true)
    .init();
```

### Correlation IDs

Request-scoped correlation IDs propagate through:
- Tauri commands → `span!` with `request_id` field
- Cloud server → extract `X-Request-Id` header, pass via `tracing::Span::current()`
- DB queries → `tracing::instrument` on Store methods

### File Rotation

```toml
# tracing-appender for file-based logging
tracing-appender = { version = "0.2", features = ["rolling-file"] }

let file_appender = tracing_appender::rolling::hourly("./logs", "oz-pos.log");
```

**Status:** ✅ `tracing` already integrated. JSON output + correlation IDs available. File rotation via `tracing-appender` configured.

## P33-4: Alert Thresholds

| Alert | Condition | Severity | Runbook |
|-------|-----------|----------|---------|
| Sync queue backlog | `sync_queue_depth > 100` | Warning | Check network, increase sync frequency |
| High error rate | `error_rate > 5% (5min)` | Critical | Check DB connectivity, roll back recent deploy |
| DB connection failure | `health_check_failures > 0` | Critical | Restart DB, verify connection pool |
| Disk usage high | `disk_used > 80%` | Warning | Purge old sync items, archive logs |
| Rate limit hits | `rate_limit_hits_total > 100 (5min)` | Warning | Check for abuse, review tenant limits |
