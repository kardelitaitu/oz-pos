## Profiling with tokio-console

tokio-console provides a live dashboard of async task graphs, resource
utilisation, and polling durations. Supported on the desktop client
(Tauri), tablet client, and headless cloud-server.

### Enabling tokio-console

Build with the `console` feature and the `tokio_unstable` cfg flag:

```bash
# Desktop client
RUSTFLAGS="--cfg tokio_unstable" cargo run --features console -p oz-pos-app

# Cloud server
RUSTFLAGS="--cfg tokio_unstable" cargo run --features console -p oz-cloud-server

# Tablet client
RUSTFLAGS="--cfg tokio_unstable" cargo run --features console -p oz-pos-tablet
```

### Launching the dashboard

Install and run `tokio-console` (requires `cargo install tokio-console`):

```bash
tokio-console
```

By default it connects to `http://127.0.0.1:6669`. The dashboard shows:

- **Tasks** — each async task with its state (running, idle, polling)
- **Resources** — async channels, mutexes, and their contention
- **Timeline** — task lifecycle over time

### CI smoke test

The cloud-server includes a `#[tokio::test]` that verifies the console
subscriber initialises without panicking when the `console` feature is
enabled.

## Flamegraph Profiling

`cargo-flamegraph` generates SVG flamegraphs of CPU hot spots. Use the
helper scripts in `scripts/` for convenient defaults:

### Helper scripts

```bash
# List available benchmark targets (Windows)
powershell -File scripts/profile.ps1 -List

# List available benchmark targets (Linux/macOS)
bash scripts/profile.sh --list
```

### Profiling benchmarks

```bash
# Windows — profile the transaction_commit benchmark
powershell -File scripts/profile.ps1 -Bench transaction_commit

# Linux/macOS
bash scripts/profile.sh --bench transaction_commit
```

Available benchmarks:
- `transaction_commit` — sale creation, checkout, line items
- `barcode_lookup` — barcode search and cache hit/miss
- `money_bench` — Money arithmetic (add, sub, mul, div, serde)
- `cart_bench` — Cart operations (add line, calculate total)

### Profiling binaries

```bash
# Windows — profile the cloud-server binary
powershell -File scripts/profile.ps1 -Binary oz-cloud-server

# Linux/macOS
bash scripts/profile.sh --bin oz-cloud-server
```

Supported binaries: `oz-pos-app`, `oz-cloud-server`, `oz-pos-tablet`,
`oz-cli`, `oz-api`.

### Profiling running processes

```bash
# Windows — attach to PID 1234 (requires Admin)
powershell -File scripts/profile.ps1 -PID 1234 -Root

# Linux/macOS — attach to PID 1234 (requires sudo)
bash scripts/profile.sh --pid 1234 --root
```

### Options

| Option | Default | Description |
|--------|---------|-------------|
| `--freq` / `-Frequency` | 997 | Sampling frequency in Hz |
| `--output` / `-Output` | auto | Output SVG path |
| `--root` / `-Root` | off | Elevate privileges (PID mode) |

### Output

Flamegraphs are saved as SVG files in the current directory:

```
flamegraph-transaction_commit-20260721-143022.svg
```

Open the SVG in any browser (Chrome, Firefox, Edge) to explore the
interactive flamegraph — hover for stack traces, zoom into hot paths.

## How to use

```bash
# Run all benchmarks
cargo bench -p oz-core

# Open HTML report
open target/criterion/report/index.html

# Compare against stored baseline
cargo install critcmp
critcmp baseline current

# Profile a hot benchmark
bash scripts/profile.sh --bench transaction_commit
```

## Regeneration

```bash
cargo bench -p oz-core
# Copy target/criterion/ to docs/benchmarks/baselines/
```
