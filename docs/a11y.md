# OZ-POS — Accessibility (a11y) Compliance

## Standards
- **WCAG 2.1 AA** (Web Content Accessibility Guidelines)
- **Apple HIG** touch targets (44×44 px minimum)
- **Material Design** touch targets (48×48 dp recommended)

## Audit Checklist

### Perceivable
- [ ] All images have `alt` text or `aria-hidden="true"`
- [ ] Colour is not the sole means of conveying information
- [ ] Text contrast ratio ≥ 4.5:1 (normal) / 3:1 (large)
- [ ] Non-text content has text alternatives
- [ ] Captions provided for audio/video content

### Operable
- [x] All interactive elements have ARIA labels
- [x] Keyboard navigation: Tab order follows visual order
- [x] Focus indicators visible on all interactive elements
- [x] Focus trap in Modal (Escape to close)
- [ ] Touch targets ≥ 44×44 px (tablet)
- [ ] No keyboard traps
- [ ] Motion/animation can be disabled via `prefers-reduced-motion`

### Understandable
- [x] Language set on `<html>` element
- [ ] Error messages are clear and suggest correction
- [ ] Labels on all form inputs
- [ ] Consistent navigation across all pages
- [ ] `@fluent/react` strings are localizable

### Robust
- [ ] ARIA roles used correctly (landmarks, widgets)
- [ ] Custom components have appropriate role mappings
- [ ] Semantic HTML used where possible (`<button>`, `<nav>`, `<table>`)

### Mobile (iOS / Android)
- [ ] Touch targets ≥ 44×44 px on all interactive elements
- [ ] VoiceOver / TalkBack labels on all controls
- [ ] Sufficient space between touchable elements (≥ 8 px)
- [ ] No horizontal scrolling at tablet breakpoints

## Testing Tools
- **axe DevTools** — automated audit in browser
- **Lighthouse** — a11y score in CI
- **NVDA / VoiceOver** — screen reader testing
- **Colour Contrast Analyser** — contrast ratio verification
